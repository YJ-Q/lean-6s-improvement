# 合成用户测试协议

> 这是合成用户测试，不等同于真实用户研究。

## 固定矩阵与顺序

先执行全部 `baseline`，再执行全部 `skill`。每个条件中，人物按 P01–P05 顺序；奇数人物的场景顺序是 S02、S01、S03，偶数人物是 S03、S02、S01。

1. P01-S02, P01-S01, P01-S03
2. P02-S03, P02-S02, P02-S01
3. P03-S02, P03-S01, P03-S03
4. P04-S03, P04-S02, P04-S01
5. P05-S02, P05-S01, P05-S03

## 条件提示词

Baseline condition:

```text
你是一个通用 AI 助手。请帮助用户处理制造现场管理问题。不要读取、引用或模拟本项目的 SKILL.md、references 或案例标注。根据用户当前提供的信息自然回答。
```

Skill condition:

```text
严格使用当前仓库 SKILL.md。分类时加载 references/classification.md；判断习惯性异常时加载 references/habitual-abnormality.md；出现“已整改/闭环”时加载 references/closure-check.md。不得使用案例期望答案直接生成结论。
```

## 会话生成规则

- 每个会话都使用新的生成上下文；两种条件不共用答案。
- 模拟用户只能揭示该场景 `hidden_facts` 中已有的事实，并遵循人物卡的 `cooperation` 行为。
- 作答助手不得看到 `expected_*` 或 `required_behavior` 字段。
- Baseline 上下文不提供 `SKILL.md` 或 `references/`；Skill 上下文只读当前场景所需规则。
- 助手给出最终操作/闭环答复时停止；最多 4 个助手轮次。
- 会话固化后再独立打分，打分不得改写会话。每个维度必须有对应证据说明。
- 保留失败与低分会话，不为了抬高结果而删除或重生成。

## 评分和解读边界

八个维度均为 0–2 分，总分仅用于本次形成性对比。不报告合成满意度、生产率、事故减少、ROI 或统计显著性。最终结论必须同时出现“合成用户测试”和“不等同于真实用户研究”。
